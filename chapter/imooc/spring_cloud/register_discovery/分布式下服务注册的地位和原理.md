# 分布式下服务注册的地位和原理

分布式系统中为什么需要服务发现？
* 简单说就是服务数量多的时候，就必然需要一个角色来管理这些服务的地址；
* 服务之间需要调用只需要去服务中心拿到某一个服务的地址即可访问



* 客户端发现：

  服务器A去服务中心拿到某一个服务的多台机器地址，通过均衡负载机制挑选一个访问，这个过程就叫客户端发现；

* 服务端发现：  

  也就是代理；服务端根据均衡负载策略挑选一个给需要访问的角色；


* 客户端发现 ： Eurek
* 服务端发现
  - nginx
  - zookeeper
  - kubernetes
